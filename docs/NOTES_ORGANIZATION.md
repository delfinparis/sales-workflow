# Notes Organization Guide

This guide explains where different types of information should be recorded in Monday.com to avoid confusion.

---

## Three Places for Notes

| Location | What Goes Here | Who Writes | When |
|----------|----------------|------------|------|
| **Updates** (activity log) | Conversation events, status changes, automated logs | Automations + Manual | Ongoing |
| **Notes** column | Manual observations, personal insights | Ana, DJ | As needed |
| **AI Meeting Notes** column | AI-generated meeting summaries | MeetGeek / Gemini | After meetings |

---

## Quick Reference: Where Do I Put This?

| Scenario | Where to Put It |
|----------|-----------------|
| Lead mentioned their kid's birthday is next week | Notes column |
| They replied "Thanks for the congrats!" | Updates (auto-logged by JustCall integration) |
| AI summary of DJ's recruiting meeting | AI Meeting Notes column |
| "This lead seems hesitant about leaving current brokerage" | Notes column |
| Status changed to Tool Offered | Updates (auto-logged) |
| DJ's prep questions for upcoming meeting | Current Tasks (auto-populated) |
| "They asked about Kale's commission structure" | Updates (quick note during call) |
| "Great call - they're definitely interested" | Notes column |

---

## Best Practices

1. **Keep Notes column concise** - It's for insights, not transcripts
2. **Use Updates for real-time notes** - During calls, jot quick notes in Updates
3. **Review AI Meeting Notes** - Always read the AI summary, but don't edit it
4. **Don't duplicate** - If it's auto-logged, don't manually write it again
5. **Use @mentions** - Tag team members in Updates when handoffs happen
